# 01 业务规格说明

## 1. 用户与角色规则

系统用户用于登录系统，不等同于读者。

用户角色包括：

- ADMIN：系统管理员
- LIBRARIAN：图书管理员
- STUDENT：学生读者

权限规则：

| 功能 | ADMIN | LIBRARIAN | STUDENT |
|---|---|---|---|
| 用户管理 | 是 | 否 | 否 |
| 图书管理 | 是 | 是 | 否 |
| 读者管理 | 是 | 是 | 否 |
| 借书 | 是 | 是 | 否 |
| 还书 | 是 | 是 | 否 |
| 查看全部借阅记录 | 是 | 是 | 否 |
| 查看个人借阅记录 | 是 | 是 | 是 |
| 查询图书 | 是 | 是 | 是 |

## 2. 图书业务规则

一本图书包含：

- 图书编号
- ISBN
- 书名
- 作者
- 出版社
- 分类
- 总数量
- 可借数量
- 状态

图书状态：

- AVAILABLE：可用
- DISABLED：停用

业务规则：

1. ISBN 可以重复，因为同一本书可能有多册。
2. 总数量不能小于 0。
3. 可借数量不能小于 0。
4. 可借数量不能大于总数量。
5. 停用图书不能被借出。
6. 删除图书时，如果存在未归还借阅记录，则不允许删除。

## 3. 读者业务规则

读者包含：

- 读者编号
- 姓名
- 学号
- 学院
- 电话
- 状态

读者状态：

- ACTIVE：正常
- DISABLED：停用

业务规则：

1. 学号唯一。
2. 停用读者不能借书。
3. 存在未归还记录的读者不允许删除。
4. 一个读者最多同时借 5 本书。

## 4. 借阅业务规则

借阅记录包含：

- 借阅编号
- 读者ID
- 图书ID
- 借出时间
- 应还时间
- 实际归还时间
- 借阅状态

借阅状态：

- BORROWED：借出未还
- RETURNED：已归还
- OVERDUE：逾期未还

业务规则：

1. 借书时，默认借阅期限为 30 天。
2. 借书成功后，图书可借数量减 1。
3. 还书成功后，图书可借数量加 1。
4. 如果当前日期超过应还日期，且图书未归还，则视为逾期。
5. 有逾期未还记录的读者不能继续借书。
6. 同一读者不能重复借阅同一本未归还图书。

## 5. 登录业务规则

1. 用户名唯一。
2. 密码必须加密存储。
3. 登录成功返回 JWT。
4. 登录失败返回统一错误信息，不暴露具体是用户名错误还是密码错误。
