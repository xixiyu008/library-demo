# 00 项目愿景：大学图书管理系统（简化版）

## 1. 项目名称

大学图书管理系统 University Library Management System，简化版。

## 2. 项目目标

开发一个前后端分离的大学图书管理系统，用于管理图书、读者、借阅和归还业务。

系统需要支持：

- 用户登录
- 角色权限控制
- 图书管理
- 读者管理
- 借书
- 还书
- 借阅记录查询
- 逾期状态判断

## 3. 系统边界

本系统是教学与练习用简化系统，不包含以下功能：

- 在线支付
- 图书采购流程
- 多校区馆藏调拨
- 复杂罚款结算
- RFID设备对接
- 微信小程序
- 移动端 App
- 大数据推荐
- Elasticsearch全文搜索

## 4. 用户角色

系统包含三类角色：

### 4.1 管理员 ADMIN

管理员可以：

- 管理用户
- 管理图书
- 管理读者
- 查看全部借阅记录

### 4.2 图书管理员 LIBRARIAN

图书管理员可以：

- 管理图书
- 办理借书
- 办理还书
- 查看借阅记录

### 4.3 学生读者 STUDENT

学生可以：

- 查询图书
- 查看自己的借阅记录
- 查看自己是否有逾期图书

## 5. 核心业务流程

### 5.1 登录流程

用户输入用户名和密码，系统校验成功后返回 JWT Token。

### 5.2 图书管理流程

管理员或图书管理员可以新增、修改、删除、查询图书。

### 5.3 借书流程

图书管理员输入学生编号和图书编号，系统检查：

- 学生是否存在
- 图书是否存在
- 图书是否还有库存
- 学生是否已经达到最大借阅数量
- 学生是否存在逾期未还记录

检查通过后，生成借阅记录，并减少图书可借数量。

### 5.4 还书流程

图书管理员输入借阅记录编号，系统检查该记录是否存在且未归还。

检查通过后，更新归还时间，并增加图书可借数量。

## 6. 成功标准

系统完成后应满足：

- 可以正常登录
- 可以完成图书 CRUD
- 可以完成读者 CRUD
- 可以完成借书和还书
- 可以查询借阅记录
- 可以识别逾期状态
- 前后端接口格式统一
- 后端分层清晰
- 代码结构适合教学讲解
