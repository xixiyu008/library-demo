# 09 代码审查清单

## 1. 架构审查

检查项：

- Controller 是否只调用 Service？
- Controller 是否没有直接调用 Mapper？
- Service 是否承担业务逻辑？
- Mapper 是否只负责 SQL？
- Entity 是否没有直接返回给前端？
- DTO/VO 是否划分清楚？

## 2. 业务规则审查

检查项：

- 借书时是否检查库存？
- 借书时是否检查读者状态？
- 借书时是否检查图书状态？
- 借书时是否检查最大借阅数量？
- 借书时是否检查逾期未还？
- 借书时是否禁止重复借同一本未归还图书？
- 还书时是否防止重复还书？
- 删除图书时是否检查未归还记录？
- 删除读者时是否检查未归还记录？

## 3. 安全审查

检查项：

- 密码是否 BCrypt 加密？
- 登录失败是否不暴露具体原因？
- 是否禁止返回 password？
- 是否所有业务接口默认需要登录？
- 是否根据角色限制接口访问？
- 是否避免日志输出敏感信息？

## 4. 事务审查

检查项：

- 借书是否加事务？
- 还书是否加事务？
- 图书库存变化和借阅记录变化是否原子完成？
- 业务异常是否能触发回滚？

## 5. API 审查

检查项：

- 是否统一返回 Result<T>？
- 错误码是否统一？
- 前后端字段名是否一致？
- API 路径是否符合 05-api-spec.md？
- 是否存在接口重复或语义混乱？

## 6. 前端审查

检查项：

- Axios 是否统一封装？
- 是否自动携带 Token？
- 路由是否做登录拦截？
- 菜单是否按角色显示？
- 页面是否没有直接硬编码后端根地址？
